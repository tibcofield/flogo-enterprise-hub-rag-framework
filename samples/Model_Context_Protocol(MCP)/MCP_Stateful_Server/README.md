## Overview

This sample demonstrates how to configure a **stateful Flogo MCP server** (`statelessServer = false`) using a realistic **multi-step loan application wizard** scenario.

A stateful MCP server issues a unique `Mcp-Session-Id` during initialization. The MCP client must include this session ID in the header of every subsequent request. This allows the server to track per-client context вЂ” such as accumulated wizard state вЂ” across multiple tool calls that belong to the same logical session.

## Stateless vs Stateful вЂ” Quick Reference

| Feature | Stateless (`statelessServer: true`) | Stateful (`statelessServer: false`) |
|---|---|---|
| `Mcp-Session-Id` issued | вќЊ No | вњ… Yes |
| Client must send session header | вќЊ No | вњ… Yes (on all subsequent requests) |
| Per-client context tracked server-side | вќЊ No | вњ… Yes |
| Safe for horizontal scaling / load balancers | вњ… Yes | вљ пёЏ Only with sticky sessions or external state store |
| Required for MCP Elicitation | вќЊ No | вњ… Yes |
| Required for MCP Sampling | вќЊ No | вњ… Yes |
| Required for MCP Logging streams | вќЊ No | вњ… Yes |
| Best for | Read-only lookups, shared data APIs, high-volume tools | Multi-step wizards, conversation context, user-specific sessions |

## The `statelessServer` Setting in Flogo

In the Flogo MCP server trigger, the `statelessServer` field is set at the **server level**:

```json
"settings": {
  "serverName": "LoanApplicationMCPServer",
  "serverType": "HTTP",
  "serverPort": "9095",
  "statelessServer": false,
  "serverEndpointPath": "/mcp"
}
```

- **`statelessServer: false`** вЂ” The server issues a globally unique, cryptographically secure `Mcp-Session-Id` in the `Mcp-Session-Id` HTTP response header during initialization. The MCP client must include this header in all subsequent requests. Requests missing the session header are rejected with `HTTP 400 Bad Request`. The server can associate any server-side state (conversation context, in-progress workflow data) with this session ID.
- **`statelessServer: true`** вЂ” No session ID is issued. See the [MCP_Stateless_Server](../MCP_Stateless_Server/README.md) sample.

## Why Stateful for Loan Application?

A loan application wizard has properties that make stateful essential:

- **Multi-step, accumulated context** вЂ” The applicant provides personal details in step 1, financial data in step 2, and the final submission in step 4 needs all of that accumulated data. In this sample, step 2 parses the JSON created by step 1 and writes back a merged session document, so later tools can review and submit the full application state. Without session continuity, each tool call would arrive at the server with no prior context.
- **Ordered workflow** вЂ” Steps must execute in sequence: `start_loan_application` в†’ `submit_financial_profile` в†’ `review_application` в†’ `submit_application`. The session guarantees that all four calls from the same agent are routed correctly and share state.
- **Per-applicant isolation** вЂ” Different applicants running concurrent applications must have completely separate session contexts. The `Mcp-Session-Id` naturally isolates each applicant's in-progress data.
- **Review-before-submit pattern** вЂ” The applicant must review accumulated state (step 3) before the server can proceed to external submission (step 4). This requires the server to have stored the earlier inputs.

### How Session State Is Stored in This Sample

The Flogo flows use **file-based state persistence** (via the Flogo Files connector `read`/`write` activities) keyed by `application_ref`. This keeps the sample self-contained and runnable without external dependencies.

For readability, the sample uses a **deterministic mock application reference** in the form `LOAN-{loan_type}-20260414`, then stores the state file as `session_{application_ref}.json`. In production, replace that mock value with a UUID or timestamp-based reference generated by your app or backend.

```
Session flow:
  start_loan_application    в†’ writes  /tmp/loan_application_sessions/session_{ref}.json
  submit_financial_profile  в†’ reads + updates same file
  review_application        в†’ reads   same file (read-only)
  submit_application        в†’ reads   same file в†’ submits в†’ generates tracking number
```

> **Production guidance:** Replace the file activities with a distributed cache (Redis), database (PostgreSQL, MySQL), or your organisation's state management service. In a load-balanced deployment, external state storage is required so any server instance can serve any request within the same session.

## Tool Summary

| Tool | Step | readOnly | destructive | idempotent | openWorld | Description |
|---|:---:|:---:|:---:|:---:|:---:|---|
| `start_loan_application` | 1 | вќЊ | вќЊ | вќЊ | вќЊ | Creates a new application session with applicant details |
| `submit_financial_profile` | 2 | вќЊ | вќЊ | вњ… | вќЊ | Adds/updates financial data in the session (safe to retry) |
| `review_application` | 3 | вњ… | вќЊ | вЂ” | вќЊ | Read-only review of all accumulated session state |
| `submit_application` | 4 | вќЊ | вќЊ | вќЊ | вњ… | Final submission to external loan origination API |

### Why `submit_financial_profile` is Idempotent

If the AI agent retries this call (e.g. due to a network timeout) with the same financial data, the session state file is overwritten with identical content вЂ” the resulting state is the same as if it had been called once. No duplicate records are created.

### Why `submit_application` is NOT Idempotent

Each call dispatches a new submission to the external loan origination API, which generates a **new unique tracking number**. Calling it twice would create two separate underwriting cases. The agent should NOT retry this tool without user confirmation.

## Getting Started

### Prerequisites

- TIBCO FlogoВ® **2.26.3** or later
- An MCP-capable client (e.g. GitHub Copilot in VS Code, Claude Desktop)

### Import the App

Import `LoanApplicationMCPServer.flogo` into VS Code using the Flogo extension.

### Configure App Properties

| Property | Default | Description |
|---|---|---|
| `FlogoMcpServer.PORT` | `9095` | HTTP port the MCP server listens on |
| `session_state_dir` | `/tmp/loan_application_sessions` | Directory for session state files. Update this for your environment. On Windows, use a path like `C:/Temp/loan_sessions`. |

### Run the App

Run `LoanApplicationMCPServer.flogo` from VS Code. The server starts at:

```
http://localhost:9095/mcp
```

### Configure Your MCP Client

**VS Code (`mcp.json`):**
```json
{
  "servers": {
    "LoanApplicationMCPServer": {
      "type": "http",
      "url": "http://localhost:9095/mcp"
    }
  }
}
```

**Claude Desktop (`claude_desktop_config.json`):**
```json
{
  "mcpServers": {
    "LoanApplicationMCPServer": {
      "command": "npx",
      "args": ["mcp-remote", "http://localhost:9095/mcp"]
    }
  }
}
```

The flows use file activities for state storage. For production:

| Component | Replace with |
|---|---|
| Session state (file write/read) | Redis / Memcached / PostgreSQL / your enterprise data store вЂ” keyed by `application_ref` |

| Load balancing | Sticky sessions (session-affinity) OR move state storage to external distributed cache so any instance can handle any session request |

## When to Use Stateful вЂ” Decision Guide

Use `statelessServer: false` when **any** of the following are true:

- вњ… The workflow has multiple steps that must share accumulated context
- вњ… Different users must have isolated, per-session data
- вњ… You need MCP Elicitation (server sends interactive forms to client mid-flow)
- вњ… You need MCP Sampling (server asks client's LLM for a completion mid-flow)
- вњ… You need persistent server-to-client SSE notification streams (MCP Logging)
- вњ… The tool sequence must be ordered and the server must enforce it

Use `statelessServer: true` (see [MCP_Stateless_Server](../MCP_Stateless_Server/README.md)) when each tool call is self-contained and requires no context from previous calls.
